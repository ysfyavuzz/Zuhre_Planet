# AI Documentation Maintenance Instructions

## Mission
The mission of the AI Documentation is to maintain a comprehensive and dynamic set of resources that guide the use and development of AI within our organization. This includes providing clear and concise information on workflows, templates, and priorities for documentation.

## Session Workflow
1. **Initialization**: Start with reviewing past documentation related to AI projects and updates.
2. **Collaboration**: Engage with team members to gather insights and updates on ongoing projects and initiatives.
3. **Documentation Update**: Make necessary updates in the documentation template as per the latest session reviews and collaborations.
4. **Review and Approval**: Present the updated documentation for review to lead members and get approvals.
5. **Publishing**: Once approved, publish the updates in the repository for team access.
6. **Feedback Loop**: Encourage team members to provide feedback on the usefulness of the documentation and suggested improvements.

## Priorities
- Ensure all AI-related documentation is up to date, with a focus on accuracy and clarity.
- Prioritize documentation for ongoing projects that impact a large number of stakeholders.
- Maintain a balance between technical detail and user accessibility.

## Rules
- Always use the latest version of the documentation template.
- Ensure proper version control for all documentation files.
- Seek input from cross-functional teams when necessary to ensure comprehensive coverage.

## Monthly Targets
- Complete reviews of all active AI projects and update documentation accordingly.
- Set aside time each month for team members to review documentation and suggest changes or improvements.
- Track all changes and updates in a changelog format for transparency and accountability.
